# Produce Low Fidelity Wireframes

## Purpose

Produce low-fidelity wireframe specifications for the MVP user flows and information architecture. Use during define before the MVP PRD is finalized.

## Entry Conditions

- Use only after user flows and IA exist. Do not use to create high-fidelity visual systems or engineering implementation details.
- Required upstream artifacts, approvals, and workflow context for this node are available.

## Required Inputs

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-inputs) — no skill-specific additions.

## Procedure

1. Map the MVP user flows to the minimum required interaction surfaces.
2. Describe the primary elements and states each surface must contain.
3. Keep the artifact low-fidelity and behavior-focused.
4. Record open UX decisions that still affect the PRD boundary.

## Permitted Tools

- `Read`, `Grep`, `Glob`

## Required Evidence

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-evidence) — no skill-specific additions.

## Output Artifact Contract

Return:

- `wireframe_specification`
- `surface_states`
- `open_ux_decisions`

## Handoff Destination

## Explicit Non-Goals

- Do not convert this into a high-fidelity visual design task.
- Do not skip interaction surfaces needed by in-scope requirements.
- Do not hide unresolved UX tradeoffs.
