# Build Interactive Prototype

## Purpose

Build a bounded interactive prototype manifest from the approved designs and flows so critical journeys can be exercised before implementation. Use during design after hi-fi and design-system work.

## Entry Conditions

- Use only after high-fidelity design and design-system artifacts exist. Do not use to replace implementation planning or usability analysis.
- Required upstream artifacts, approvals, and workflow context for this node are available.

## Required Inputs

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-inputs) — no skill-specific additions.

## Procedure

1. Define the critical journeys that must be exercisable.
2. Build a prototype manifest that maps those journeys to prototype surfaces.
3. Call out prototype gaps and limits explicitly.
4. Keep the prototype bounded to MVP decisions.

- `Read`, `Grep`, `Glob`

- `prototype_limits`

- Do not skip critical journeys.
- Do not hide prototype limitations.
