# Create High Fidelity Interface

## Purpose

Convert approved wireframes and flows into a bounded high-fidelity interface specification with required states and breakpoints. Use at the start of the design phase before design-system definition.

## Entry Conditions

- Use only after wireframes exist. Do not use to redefine product scope or to start frontend implementation.
- Required upstream artifacts, approvals, and workflow context for this node are available.

## Required Inputs

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-inputs) — no skill-specific additions.

## Procedure

1. Translate wireframes and flows into high-fidelity screen and component specifications.
2. Include required responsive states and major interaction states.
3. Keep the artifact traceable to the approved UX and PRD artifacts.
4. Record the biggest unresolved visual or behavior ambiguity.

## Permitted Tools

- `Read`, `Grep`, `Glob`

- `represented_states`
- `design_risks`

## Explicit Non-Goals

- Do not add features absent from the approved scope.
- Do not omit required states to make the design look simpler.
- Do not hide breakpoint or accessibility assumptions.
