# Design Information Architecture

## Purpose

Define the navigation and content relationships needed to support approved user flows. Use during define after user flows and before low-fidelity wireframes.

## Entry Conditions

- Use only after user flows exist. Do not use to create visual hierarchy or implementation structure.
- Required upstream artifacts, approvals, and workflow context for this node are available.

## Required Inputs

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-inputs) — no skill-specific additions.

## Procedure

1. Translate the user flows into screens, navigation, and content relationships.
2. Keep the structure minimal for the MVP.
3. Make dependencies between surfaces explicit.
4. Record the IA ambiguity most likely to affect scope.

- `Read`, `Grep`, `Glob`

## Explicit Non-Goals

- Do not invent scope beyond the prioritized capabilities.
- Do not hide navigation or content ambiguities.
