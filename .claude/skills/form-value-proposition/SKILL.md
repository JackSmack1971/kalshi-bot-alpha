# Form Value Proposition

## Purpose

Form one explicit value proposition from target users, jobs-to-be-done, alternatives, and differentiated outcomes. Use during discovery after target users and market alternatives are clear enough to compare.

## Entry Conditions

- Use only in discovery. Do not use to approve the final core problem or expand the MVP scope.
- Required upstream artifacts, approvals, and workflow context for this node are available.

## Required Inputs

See [Idea-to-MVP Common Skill Sections](../_shared/idea-to-mvp-common-sections.md#required-inputs) — no skill-specific additions.

## Procedure

1. Name the target user and the job being improved.
2. State the desired outcome and the current alternative.
3. Explain the specific differentiated value, or mark it as a hypothesis.
4. Record the biggest assumption that still needs testing.

## Permitted Tools

- `Read`, `Grep`, `Glob`

- `current_alternative`
- `differentiation`
- `strategic_assumptions`

## Explicit Non-Goals

- Do not write marketing copy instead of a bounded proposition.
- Do not omit the current alternative.
- Do not claim differentiation without evidence or a stated assumption.
