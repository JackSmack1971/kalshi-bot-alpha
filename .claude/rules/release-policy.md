# Release Policy Rules

- Keep deployment evidence, analytics readiness, release authorization, and post-launch learning as separate decisions.
- Analytics must be validated before launch metrics are used for product conclusions.
- Release status must distinguish ready, conditional, and blocked outcomes explicitly.
- Post-launch review must normalize signals against hypotheses instead of listing raw telemetry alone.
- Next-iteration planning must end with one explicit decision and any required human approval.
